# v0.10.0 — Web Dashboard

Released: TBD

## Overview

This release introduces a built-in web dashboard that provides a browser-based interface for viewing SMPStats data. The dashboard supports configurable public and admin access, allowing server operators to share stats publicly while keeping sensitive data protected.

## New Features

### Web Dashboard Server
- **Separate HTTP server** on configurable port (default 8080)
- **Static file serving** for HTML/CSS/JS assets
- **CORS support** for cross-origin requests
- **Lifecycle integration** with plugin enable/disable/reload

### Public Access (No Authentication)
- **Online Players** — Real-time count and list of online players
- **Leaderboards** — Top players for playtime, blocks, kills, deaths, distance
- **Recent Moments** — Latest notable events with timestamps and details
- **Server Stats** — Aggregated totals across all players

### Admin Access (Password Required)
- **Full Player Stats** — Complete statistics for all players
- **Heatmap Data** — Mining, death, and position heatmaps by world
- **Timeline Data** — Historical daily snapshots per player
- **Social Data** — Player pairs and interaction metrics

### Configuration
New `dashboard` section in `config.yml`:
```yaml
dashboard:
  enabled: true
  bind_address: "0.0.0.0"
  port: 8080
  public:
    enabled: true
    show_online_players: true
    show_leaderboards: true
    show_recent_moments: true
    show_server_stats: true
  admin:
    enabled: true
    password: "ChangeThisAdminPassword"
    session_timeout_minutes: 60
```

### Frontend
- **Dark theme** with Minecraft-inspired color palette
- **Responsive design** for desktop and mobile
- **Auto-refresh** every 30 seconds for live data
- **Admin login** with password authentication
- **Tabbed navigation** for different data views

## Technical Changes

### New Classes
- `de.nurrobin.smpstats.dashboard.WebDashboardServer` — Main dashboard HTTP server
- Dashboard HTTP handlers for each endpoint type

### Updated Classes
- `Settings` — Added `DashboardSettings`, `PublicSettings`, `AdminSettings` records
- `SMPStats` — Dashboard server lifecycle management, config version 6
- `ApiServer` — Uses configurable bind address

### New Resources
- `resources/dashboard/index.html` — Dashboard HTML page
- `resources/dashboard/css/style.css` — Dashboard styles
- `resources/dashboard/js/app.js` — Dashboard JavaScript application

### Configuration Version
- Config version bumped from 5 to 6
- Automatic migration adds `dashboard` section with defaults

## Breaking Changes

None. Existing configurations will be automatically upgraded.

## Migration Notes

1. After upgrading, the dashboard will be enabled by default on port 8080
2. Change `dashboard.admin_key` from the default value for security
3. Adjust `dashboard.public.*` settings to control what's publicly visible
4. Use a reverse proxy with TLS for production deployments
